# 私有协议小团队IM系统

## 产品形态
1. 整体采用BS架构，服务端在本机，客户端在手机浏览器上（默认Chrome移动版）
2. 各个客户端之间通过服务端做消息同步，同步可以接受10s以内的延迟
3. 服务端支持最多50个用户注册，不超过20个客户端的并发访问
4. 客户端可以发送文字、语音、图片进行1对1聊天，不支持群聊，但可以分享对话

## 技术栈
1. 后端技术栈依产品形态选择，尽量轻型、尽量节省硬件资源
2. 通信协议要支持压缩，尽量减少传输数据量
3. 前端界面朴素实用，不加载非必要的资源

## 要求
1. 使用项目级开发和运行环境，不要污染公共环境
2. 前后端要采用结构化设计，分整体设计和模块（及子模块）设计，先整体再部分。所有设计要归档，如果实现中改变了设计要立即更新文档
3. 模块设计完毕后，要把对外接口单独归档，如果后续实现过程中修改了对外接口的定义，要立即更新文档
4. 模块间边界清晰，实现模块时只需阅读自身设计文档和依赖模块的外部接口文档，无需阅读其他文档
5. 推荐使用TDD流程进行模块开发，即使不走TDD流程也要保证模块实现后有单元测试环节，通过单元测试才能开发下一个模块
